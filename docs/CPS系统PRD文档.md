## 2. 产品概述

### 2.1 产品背景

电商CPS（Cost Per Sale）联盟是当前主流的商品推广模式。淘宝联盟、京东联盟、拼多多联盟等平台为推广者提供佣金激励，当消费者通过推广链接完成购买后，推广者可获得销售佣金。

市场上大量消费者有"购物省钱"的明确诉求，同时缺少便捷途径获取各平台优惠券和返利信息。本产品将多个CPS联盟平台聚合，为消费者提供一站式的返利查询、比价和提现服务，同时为运营方提供可持续的佣金收益模型。

### 2.2 产品定位

**一站式多平台CPS返利查询与导购系统**：聚合淘宝、京东、拼多多等主流电商平台的CPS联盟能力，为会员提供返利查询、跨平台比价、推广链接生成和返利提现等服务。

### 2.3 目标用户

| 用户类型 | 画像描述 | 核心诉求 |
|----------|----------|----------|
| **普通消费者** | 有网购习惯，追求性价比 | 购物省钱、获取优惠券、返利到账 |
| **返利达人** | 熟悉CPS模式，主动分享赚钱 | 高返利比例、邀请分润、便捷提现 |
| **平台运营者** | 系统管理员、运营人员 | 平台管理、返利规则配置、数据分析 |

### 2.4 产品目标

| 目标 | 衡量指标 |
|------|----------|
| 用户获取 | 注册会员数、日活/月活用户数 |
| 用户转化 | 查询到下单的转化率 |
| 用户留存 | 次日留存率、7日留存率、30日留存率 |
| 营收增长 | 月度佣金收入、平台净利润 |
| 用户满意度 | 提现成功率、返利到账时效 |

## 4. 业务流程

### 4.1 核心业务流程总览

```
会员注册/登录
    │
    ▼
商品查询（关键词/链接/口令）
    │
    ├── 单平台查询 ──→ 展示商品列表（含预估返利）
    │
    └── 多平台比价 ──→ 展示跨平台比价结果
           │
           ▼
    选择商品，生成推广链接/口令
           │
           ▼
    会员复制链接/口令，到对应电商APP下单
           │
           ▼
    系统定时同步订单（每5分钟）
           │
           ├── 订单归因匹配 ──→ 关联到会员
           │
           ▼
    订单状态追踪（付款→收货→结算）
           │
           ▼
    平台确认佣金结算
           │
           ▼
    系统计算会员返利（按等级比例）
           │
           ▼
    返利入账到会员钱包
           │
           ▼
    会员申请提现 → 审核 → 打款到账
```

### 4.2 商品查询流程

```
用户输入内容
    │
    ▼
智能识别输入类型
    │
    ├── 匹配到URL特征 ──→ 识别平台 ──→ 提取商品ID ──→ 调用单平台API
    │     taobao.com / tmall.com → 淘宝
    │     jd.com                 → 京东
    │     pinduoduo.com          → 拼多多
    │
    ├── 匹配到口令特征 ──→ 解析口令 ──→ 获取商品信息
    │     ￥...￥               → 淘口令
    │
    └── 纯文本关键词 ──→ 并发查询所有启用平台
          │
          ▼
    聚合各平台返回结果
          │
          ▼
    根据当前会员等级计算预估返利金额
          │
          ▼
    按指定排序方式排序（价格/返利/销量）
          │
          ▼
    返回统一格式的商品列表
```

### 4.3 推广链接生成流程

```
会员选择某商品
    │
    ▼
确定平台和商品ID
    │
    ▼
获取当前会员的推广位（PID）
    │
    ├── 会员有专属推广位 → 使用专属PID
    └── 无专属推广位 → 使用平台默认PID
          │
          ▼
注入用户归因参数
    │
    ├── 淘宝: adzone_id + external_info
    ├── 京东: subUnionId = 会员标识映射
    └── 拼多多: custom_parameters = {"uid":"会员ID"}
          │
          ▼
调用平台转链API
          │
          ▼
返回推广链接 + 口令（淘宝）
          │
          ▼
记录转链日志
```

### 4.4 订单同步与结算流程

```
定时任务触发（每5分钟）
    │
    ▼
遍历所有启用的CPS平台
    │
    ▼
调用各平台订单查询API（增量查询）
    │
    ▼
解析返回的订单数据
    │
    ├── 新订单 → 解析归因参数 → 匹配会员 → 入库
    │
    └── 已有订单 → 检查状态变化 → 更新状态
          │
          ├── 变为"已结算" → 触发返利结算流程
          │     │
          │     ▼
          │   计算可分配佣金（佣金 - 平台服务费）
          │     │
          │     ▼
          │   查询会员返利比例（优先级：个人 > 等级+平台 > 等级 > 平台 > 全局）
          │     │
          │     ▼
          │   计算返利金额 = 可分配佣金 x 返利比例
          │     │
          │     ▼
          │   增加会员钱包余额（调用PayWalletService）
          │     │
          │     ▼
          │   创建返利记录 → 通知会员
          │
          └── 变为"已退款" → 触发返利扣回流程
                │
                ▼
              返利已入账? → 扣减钱包余额 → 创建扣回记录 → 通知会员
              返利未入账? → 取消待结算记录
```

### 4.5 提现流程

```
会员发起提现申请
    │
    ▼
校验提现条件
    │
    ├── 余额是否充足
    ├── 是否达到最低提现金额（默认¥1）
    ├── 是否超过每日提现次数（默认3次）
    ├── 是否超过单次提现上限（默认¥5000）
    └── 是否在黑名单中
          │
          ▼
创建提现申请记录
          │
          ▼
审核判断
    │
    ├── 金额 ≤ 审核阈值（默认¥500） → 自动审核通过
    │
    └── 金额 > 审核阈值 → 进入人工审核队列
          │                    │
          │               运营审核
          │                    │
          │            ┌───────┴───────┐
          │            ▼               ▼
          │        审核通过         审核驳回
          │            │               │
          ▼            ▼               ▼
    调用转账API      同上           返还余额
    （支付宝/微信）                 通知会员
          │
          ├── 成功 → 扣减余额 → 记录流水 → 通知会员
          └── 失败 → 返还余额 → 通知会员 → 标记异常
```

## 6. 功能详细设计

### 6.1 商品搜索（M-001）

**页面**: 首页搜索框 + 搜索结果页

**交互流程**:

1. 会员在首页搜索框输入内容，点击搜索按钮
2. 系统判断输入类型（关键词/链接/口令）
3. 加载中展示骨架屏，单平台结果先到先展示
4. 搜索结果页按卡片列表展示商品

**搜索结果卡片信息**:

```
┌─────────────────────────────────────────┐
│ [商品图片]  商品标题（最多两行）           │
│             ┌──────┐                     │
│             │ 淘宝  │  店铺名称            │
│             └──────┘                     │
│                                         │
│  ¥89.00（原价）→ 券后 ¥59.00             │
│  优惠券 ¥30    月销 2.3万                 │
│                                         │
│  ┌─────────────────────────┐            │
│  │  预估返利 ¥5.90          │            │
│  └─────────────────────────┘            │
│                          [立即购买]       │
└─────────────────────────────────────────┘
```

**筛选与排序**:
- 筛选: 平台（全部/淘宝/京东/拼多多）、有无优惠券、价格区间
- 排序: 综合（默认）、价格升序、价格降序、销量降序、返利降序

**异常处理**:
- 某平台API超时：展示已返回平台的结果，超时平台显示"暂时无法查询"
- 全部平台无结果：展示"未找到相关商品，请换个关键词试试"
- 未登录查询：正常展示商品，返利金额显示"登录查看返利"

### 6.2 多平台比价（M-003）

**页面**: 比价结果页（搜索结果页的Tab切换或独立入口）

**交互流程**:

1. 会员输入关键词后选择"比价"模式
2. 系统并发查询所有启用平台
3. 以对比表格形式展示结果

**比价卡片**:

```
┌─────────────────────────────────────────┐
│  "iPhone 16 手机壳"  比价结果             │
│                                         │
│  ★ 最省钱: 拼多多                        │
├─────────┬────────┬────────┬─────────────┤
│  平台    │ 券后价  │ 返利   │ 实付         │
├─────────┼────────┼────────┼─────────────┤
│  拼多多   │ ¥24    │ ¥2.4  │ ¥21.6  最低  │
│  淘宝    │ ¥25    │ ¥3.0  │ ¥22.0       │
│  京东    │ ¥29    │ ¥1.5  │ ¥27.5       │
└─────────┴────────┴────────┴─────────────┘
  点击任一行进入对应平台商品详情
```

**排序模式**（可切换）:
- 实付最低（默认）: 券后价 - 返利
- 券后价最低
- 返利最高

### 6.3 推广链接生成（M-005）

**页面**: 商品详情页底部操作栏

**交互流程**:

1. 会员在商品详情页点击"立即购买"或"获取链接"
2. 系统调用平台转链API生成推广链接
3. 弹出底部面板展示推广内容

**底部面板内容**:

```
┌─────────────────────────────────────────┐
│  获取推广链接                             │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  ￥dSf8JcGh￥                    │    │
│  │  复制口令打开淘宝APP即可购买        │    │
│  └─────────────────────────────────┘    │
│                                         │
│  [复制口令]    [复制链接]    [分享海报]    │
│                                         │
│  预估返利 ¥5.90（购买确认收货后到账）      │
└─────────────────────────────────────────┘
```

- 结构化商品信息列表
- AI 优化的推荐理由和比价分析

## 7. 返利规则说明

### 7.1 佣金计算规则

```
               商品实付金额（买家实际支付）
                       │
                       ▼
              商品佣金 = 实付金额 x 佣金比例
                       │
                       ▼
              平台服务费 = 商品佣金 x 平台费率（如10%）
                       │
                       ▼
              可分配佣金 = 商品佣金 - 平台服务费
                       │
              ┌────────┴────────┐
              ▼                 ▼
     会员返利                 平台利润
  = 可分配佣金 x 返利比例   = 可分配佣金 - 会员返利
```

### 7.2 返利金额展示规则

| 场景 | 展示内容 |
|------|----------|
| 已登录 + 商品搜索结果 | "预估返利 ¥X.XX" |
| 已登录 + 商品详情页 | "预估返利 ¥X.XX（确认收货后到账）" |
| 未登录 + 商品搜索结果 | "登录查看返利" |
| 返利金额 < ¥0.01 | "暂无返利" |
| 已登录 + 比价页 | 显示各平台的预估返利金额 |

### 7.3 返利到账时间规则

| 环节 | 时间 | 说明 |
|------|------|------|
| 下单到追踪 | 5~30分钟 | 系统定时同步订单 |
| 追踪到结算 | 取决于平台 | 淘宝日结、京东月结、拼多多~15工作日 |
| 结算到入账 | 0~24小时 | 系统配置的入账延迟 |
| 入账到可提现 | 立即 | 入账即可提现 |

### 8.1 等级体系

复用 `yudao-module-member` 现有等级体系，CPS订单为会员贡献经验值。

| 等级 | 升级所需经验值 | CPS返利比例 | 每笔CPS订单奖励经验值 |
|------|---------------|-------------|----------------------|

| 铂金会员 | 20,000 | 80% | +50 |
| 钻石会员 | 特邀/付费 | 95% | +50 |

### 8.2 等级变动对返利的影响

- **升级**: 新订单按新等级比例计算返利
- **降级**: 新订单按新等级比例计算返利
- **已有未结算订单**: 按下单时的等级比例计算，不受后续等级变化影响

## 11. 风控策略

- 黑名单用户：冻结返利入账、禁止提现

## 14. 技术约束

### 14.1 基础框架

- 后端: ruoyi-vue-pro (Spring Boot 3.x + MyBatis Plus)
- 前端管理后台: Vue3 + Element Plus（复用现有管理后台）
- 前端会员端: UniApp 或 H5（Vue3）
- 数据库: MySQL 8.0+
- 缓存: Redis 6.0+
- 定时任务: 复用 yudao-spring-boot-starter-job

### 14.2 性能要求

| 指标 | 要求 |
|------|------|
| 单平台搜索 | < 2秒（P99） |
| 多平台比价 | < 5秒（P99） |
| 转链生成 | < 1秒 |
| 订单同步延迟 | < 30分钟 |
| 返利入账延迟 | 平台结算后 24小时内 |

### 14.3 数据安全

- CPS平台密钥 AES-256 加密存储
- 会员数据隔离（只能查看自己的数据）
- 敏感操作记录审计日志
- 提现金额二次校验

## 附录B: 术语表

| 术语 | 说明 |
|------|------|
| CPS | Cost Per Sale，按销售计费的推广模式 |
| 佣金 | CPS平台支付给推广者的销售提成 |
| 返利 | 系统将佣金的一部分返还给下单会员 |
| PID | 推广位标识，用于追踪订单来源 |
| 转链 | 将普通商品链接转换为含推广追踪参数的链接 |
| 淘口令 | 淘宝/天猫的商品口令，在APP中可自动解析 |
| 归因 | 将CPS订单关联到具体会员的过程 |
| 可分配佣金 | 佣金扣除平台技术服务费后，可用于分配的部分 |
| 券后价 | 使用优惠券后的商品实付价格 |
| 实付价 | 券后价减去预估返利后的实际花费 |
